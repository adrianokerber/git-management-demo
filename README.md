# git-management-demo
Demonstração de gerenciamento de repositório comparando dois casos.

Este repositório serve apenas como página central de referência para os estudos de caso abaixo, demonstrando boas e más praticas de gerenciamento de alterações.

Seguem os dois casos:
- [Caso Ruim](https://github.com/adrianokerber/git-management-demo-bad-case)
- [Caso Bom](https://github.com/adrianokerber/git-management-demo-good-case)

Ambos os casos demonstram o dia a dia de um time onde um deles tem a política de rastreio de mudanças bem definida e focado na baixa carga cognitiva, sendo o outro orientado ao caos onde os commits não representam muita coisa e o histórico mais atrapalha do que ajuda quando há algum problema.

Sinta-se a vontade para analisar ambos os casos e tirar suas próprias conclusões. Estes modelos são baseados em experiências passadas em ambiente de trabalho, ambos os casos representam cenário vivenciados e de forma geral são divisores de águas entre um time mais iniciante e um time de alta performance.
